ProofOps demo journey

One beautiful journey: from messy file to audit pack.

Open demo page
Upload a document

Start with a file such as Supplier_COA_Batch_118.pdf.

Register the evidence

Show object ID, file name, evidence type, source, uploader, timestamp, SHA-256 fingerprint, and review state.

Add supporting evidence

Attach a screenshot, spreadsheet, operator note, deviation comment, or AI-assisted summary.

Seal the bundle

Display the bundle ID, canonical hash, manifest, included files, and provenance event chain.

Review the record

Use public states: Raw, Witnessed, Pending Review, Approved, Rejected, Quarantined, and Exported.

Export the proof pack

Deliver a PDF audit report, ZIP evidence bundle, JSON manifest, provenance log, and reviewer decision record.

How Fieldgrade handles evidence

Files are captured, records are fingerprinted, events are chained, review status is explicit, and exports are reproducible.

Human review by design

Fieldgrade separates capture, analysis, review, and approval. Evidence may be ingested automatically, but authority remains explicit.

Local-first by default

Accountable AI use

Fieldgrade does not ask you to trust AI. It gives you the record structure to govern AI.
