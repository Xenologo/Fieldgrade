Local-first evidence governance.

Responsible AI-use

What this page says publicly

Local-first model

Evidence sealing

Data handling

Use exportable bundles, customer-controlled retention, and optional deployment rather than forced cloud dependence.

AI accountability

Record prompts, outputs, model context, reviewer decisions, and human approval status.

What this page avoids

Deterministic manifests, cryptographic hashes, replayable evidence bundles, local control, and explicit review boundaries.

Do not overexpose

Private key handling paths, deployment secrets, admin endpoints, internal trust-boundary maps, and local endpoint configuration examples.
