One public journey.

The public demo should show one beautiful path: upload a document, register it, add support, seal the bundle, review the record, and export the proof pack.

Demo flow

Use simple labels and visible proof at every step.

Upload a document

Example: Supplier_COA_Batch_118.pdf.

Register the evidence

Visible fields: object ID, file name, evidence type, source, uploader or operator, timestamp, SHA-256 fingerprint, and review state.

Add supporting evidence

Example additions: screenshot, spreadsheet, operator note, deviation comment, AI-assisted summary.

Seal the bundle

Visible outputs: bundle ID, canonical hash, manifest, included files, provenance event chain.

Review the record

Visible states: Raw, Witnessed, Pending Review, Approved, Rejected, Quarantined, Exported.

Export the proof pack

Outputs: PDF audit report, ZIP evidence bundle, JSON manifest, provenance log, reviewer decision record.
