# Fieldgrade pilot intake

Use this intake before quoting, scoping, or scheduling a Fieldgrade ProofOps Setup engagement.

## Core intake fields

- **Organisation / project name**
- **Sector**
- **Primary evidence problem**
- **Current system** — Excel / SharePoint / QMS / ERP / email
- **Desired workflow** — ProofOps / GovAI / FoodQA
- **Sample evidence types**
- **Data sensitivity**
- **Local-only requirement**
- **Deployment preference** — workstation / single host / internal VM
- **Desired proof-pack output**

## Qualification prompts

### Evidence volume

- Approximate number of files or records per week
- Typical file formats
- Need for batch imports or one-by-one capture

### Review model

- Who captures evidence
- Who reviews it
- Who signs off decisions
- Whether AI-assisted work must be shown separately

### Security and handling

- Whether internet egress is restricted
- Whether external AI endpoints are prohibited
- Whether backups must remain customer-managed
- Whether screenshots or demo data must be anonymised

### Pilot success signal

A pilot is qualified when the buyer can point to one workflow that should end in a proof pack with explicit reviewer decisions.

## First deliverable checklist

- one installed workspace
- one branded workspace context
- one workflow pack
- one export template
- one demo dataset
- one guided training session
- one agreed proof-pack example
