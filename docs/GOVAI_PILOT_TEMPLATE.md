# GovAI pilot workflow template

## Target buyer

- small AI consultancy
- SME using AI internally
- public-sector-adjacent AI pilot
- technical report writer using LLMs

## Minimum workflow objects

1. **AI system register**
   - system name
   - owner
   - purpose
   - affected users
   - deployment status
2. **Prompt/output evidence form**
   - prompt text or reference
   - output reference
   - evidence file links
   - reviewer note
3. **Model/version field**
   - model family
   - model or endpoint version
   - runtime context
4. **Risk note**
   - principal risk
   - mitigations
   - open questions
5. **Human review gate**
   - reviewer
   - review date
   - review state
6. **Approval decision**
   - approved / rejected / needs changes
   - rationale
7. **Public summary export**
   - plain-language description
   - safeguards noted
   - decision summary

## Proof-pack focus

GovAI pilots should demonstrate that AI-assisted work is recorded, challenged, reviewed, and exported with human ownership intact.
