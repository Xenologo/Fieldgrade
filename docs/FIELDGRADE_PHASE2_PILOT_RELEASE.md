# Fieldgrade Phase 2 — Pilot Release and ProofOps Commercialisation

> Convert Fieldgrade from an alpha-grade evidence-governance repo into a founder-led pilot product with release artifacts, reproducible installation, proof-pack demos, setup intake, and first paid ProofOps packaging.

## Phase outcome

The near-term win for Fieldgrade is not more architectural expansion. It is a pilot package that one person can install, understand, test with sample workflows, and use to request a paid setup.

## Priority sequence

1. **Alpha release execution** — publish the `v0.9.0-alpha` release packet and release notes.
2. **Reproducible install hardening** — lock dependencies and validate lock freshness in CI.
3. **Demo-to-sale conversion** — make setup, contact, and pilot intake obvious.
4. **Sales collateral** — screenshots, walkthrough GIF, and proof-pack PDFs.
5. **Pilot templates** — GovAI and FoodQA field-ready starter workflows.
6. **Pilot trust packet** — security, data boundaries, and AI-use disclaimers.
7. **Public alpha launch packet** — post, buyer email, founder script, and announcement copy.

## Milestone

- **v0.9.0-alpha Pilot Release**

## Explicit non-priorities

- more speculative architecture
- more product variants
- ERP or SAP adapter work
- multi-tenant SaaS expansion
- complex role hierarchy work
- full QMS claims

## Working promise

Fieldgrade should now look like a calm, installable, founder-led pilot product for evidence governance, proof-pack export, and human-reviewed AI accountability.
